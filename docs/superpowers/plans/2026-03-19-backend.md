# CozyPup Backend Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the FastAPI backend with LiteLLM-powered agents, PostgreSQL database, auth, and all API endpoints needed by the frontend.

**Architecture:** FastAPI + SQLAlchemy async + LiteLLM for model abstraction. Hand-written agents (no LangChain). SSE streaming for chat. APNs for push notifications.

**Tech Stack:** Python 3.12+, FastAPI, LiteLLM, PostgreSQL, SQLAlchemy, Alembic, sse-starlette

**Spec:** `docs/superpowers/specs/2026-03-17-petcare-agent-design.md`
**Design decisions:** See memory `project_spec_decisions.md`

- `backend/pyproject.toml` — dependencies defined

- `backend/app/routers/chat.py` — chat router shell (exists but needs full implementation)

- [ ] **Step 1: Write emergency detection**

```python
EMERGENCY_KEYWORDS = [
    "seizure", "convulsion", "poison", "toxic", "choking",
    "bleeding", "blood", "collapse", "unconscious",
    "difficulty breathing", "not breathing", "swallowed",
    "hit by car", "broken bone", "fracture",
]

def detect_emergency(message: str) -> bool:
    lower = message.lower()
    return any(kw in lower for kw in EMERGENCY_KEYWORDS)
```

### Task 7: Chat Agent (Streaming SSE)

**Files:**
- Create: `backend/app/agents/chat_agent.py`
- Modify: `backend/app/routers/chat.py`
- Test: `backend/tests/test_chat.py`

- [ ] **Step 1: Write Chat Agent**

`backend/app/agents/chat_agent.py`:
- Takes message + context + pet profiles
- Streams via LiteLLM async streaming (strong model)
- Appends disclaimer to health consultation replies
- Function calling tools:
  - `create_calendar_event(pet_id, event_date, title, category, ...)` — auto-records
  - `query_calendar_events(pet_id, date_range, category)` — queries past events
- Multi-pet resolution: infers which pet from context

- [ ] **Step 2: Write chat router**

`POST /api/v1/chat`:
1. Run emergency detection (non-blocking)
2. Run LLM Router → get intent
3. Dispatch to correct agent
4. Stream SSE events: `token`, `card`, `emergency`, `done`
5. Save user message + assistant response to chats table
6. Manage sessions (create new if none for today)

`GET /api/v1/chat/history`:
- Paginated by session
- Returns messages with cards

Use `sse-starlette` for SSE.
