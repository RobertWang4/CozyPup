# CozyPup Backend Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the FastAPI backend — 先完成所有 REST API 接口，最后再做 AI agent 逻辑。

**Architecture:** FastAPI + SQLAlchemy async + LiteLLM for model abstraction. Hand-written agents (no LangChain). SSE streaming for chat. APNs for push notifications.

**Tech Stack:** Python 3.12+, FastAPI, LiteLLM, PostgreSQL, SQLAlchemy, Alembic, sse-starlette

**Spec:** `docs/superpowers/specs/2026-03-17-petcare-agent-design.md`
**Design decisions:** See memory `project_spec_decisions.md`

### Already Exists

- `backend/pyproject.toml` — dependencies defined (FastAPI, uvicorn, sse-starlette)
- `backend/app/main.py` — FastAPI app with CORS, health endpoint, debug middleware
- `backend/app/routers/chat.py` — SSE stub with canned responses (to be replaced)
- `backend/app/agents/` — empty shell (base.py, __init__.py)
- `backend/app/debug/` — error capture, logging, correlation middleware

### Frontend Status (已完成)

前端（SwiftUI native）已完成全部 UI 和本地数据层：
- **Auth**: 模拟登录（Apple/Google），免责声明，宠物引导页
- **Chat**: SSE 流式客户端 (`ChatService.swift`)，支持 token/card/emergency/done 事件
- **Calendar**: 月历视图、事件 CRUD、日期筛选、宠物筛选
- **Settings**: 宠物管理 CRUD、通知偏好、法律页面
- **Services**: 语音输入（SpeechService）、定位（LocationService）
- **持久化**: 全部使用 UserDefaults（后续需切换到 API）

**前端 ChatService 期望的 SSE 格式：**
```
POST /api/v1/chat
Body: { message, session_id?, location?: { lat, lng } }
Events: token({text}), card(CardData), emergency({message, action}), done({intent, session_id})
```

**注意**: 前端目前没有发 auth header，集成阶段需要加。


Each creates or finds user by email, returns `{ access_token, refresh_token, user_id }`.

### Task 9: Chat SSE Endpoint (Basic — No Agent Logic)

**Files:**

把现有的 canned stub 替换为真实的 SSE endpoint，但先不接 agent 逻辑，直接 proxy 到 LLM：

- [ ] **Step 1: Rewrite chat endpoint**

```
POST /api/v1/chat
```

流程（Phase 2 简化版）：
1. 接收 message + session_id + location
2. 找到或创建今日 session
3. 保存 user message 到 chats 表
4. 直接调用 LiteLLM streaming（strong model），带基础 system prompt + pet context
5. 流式返回 SSE events: token → done
6. 保存 assistant response 到 chats 表

暂时不做：emergency detection、intent routing、function calling、card generation。这些留给 Phase 3。

### Implementation Order Rationale

1. **先做接口、后做智能体**：前端已经完成，需要尽快有可对接的 API。Task 9 提供一个能用的基础 chat SSE（直接 proxy LLM），前端可以立即连通测试。
2. **Agent 逻辑是增量叠加的**：每个 agent（emergency → router → chat → summary → map → email）逐步给 chat endpoint 加能力，不影响已有接口的稳定性。
3. **Push notifications 最后做**：依赖 reminders 数据 + device tokens，且需要 Apple Developer 证书配置。
