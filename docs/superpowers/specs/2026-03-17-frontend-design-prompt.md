# PetCare Agent — Frontend Design Prompt

你正在为一款宠物健康管家 App 设计前端界面。这不是一个宠物聊天机器人，而是一个有记忆的宠物管家——聊天是唯一的输入方式，日历/时间线是数据沉淀的载体。

## 技术栈

React + Capacitor（打包 iOS），纯 Web 技术实现，iOS 端体验必须 100% 适配原生级手感。

## App 结构

极简，只有一个主页面——聊天页。没有底部 Tab，没有多页面导航。

## 页面布局

```
┌──────────────────────────────┐
│ 📅                        ⚙️ │
│                              │
│       对话流（SSE 流式）      │
│                              │
│   ┌──────────────────────┐   │
│   │ AI建议仅供参考        │   │
│   └──────────────────────┘   │
│                              │
│  ┌────────────────────────┐  │
│  │  输入消息...        🎤  │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

### 左上角 📅 图标

点击或左滑拉出日历抽屉：
- 月视图，日期上有彩色圆点标记不同宠物的事件
- 点击日期展开当天事件列表
- 事件可内联编辑/删除
- 顶部可按宠物筛选
- 颜色从宠物档案（pets.color）中派生

### 右上角 ⚙️ 图标

点击或右滑拉出设置抽屉：
- 宠物管理（增删改宠物档案：名字、品种、生日、体重、头像）
- 账号信息
- 通知设置
- 隐私政策 / 免责声明
- 登出

### 主体：聊天界面

- 纯聊天界面，标准气泡式对话流
- AI 回复通过 SSE 流式输出，逐字显示
- 没有宠物切换器——不分宠物聊天，所有对话在一个聊天框内，AI 自动从上下文判断用户在说哪只宠物

### 底部输入区

- 文本输入框 + 语音输入按钮（调用 iOS 系统键盘语音，立即转为文字，App 本身不处理语音）
- 输入框上方常驻免责提示条："AI 建议仅供参考，紧急情况请就医"

## 聊天流内的特殊元素

### 1. 结构化卡片（Record）

AI 记录健康数据后，在回复中嵌入卡片，如：
> 已记录：3/17 豆豆 呕吐 黄水

卡片可点击跳转到日历对应条目。

### 2. 地图推荐卡片（Map）

Map Agent 返回的地点列表以卡片形式展示，包含名称、距离、评分。

### 3. 邮件草稿卡片（Email）

Email Agent 生成的预约邮件以可复制/分享的卡片展示，用户可在聊天中要求修改。

### 4. 急症提示横幅（Emergency）

检测到紧急关键词时，聊天流顶部弹出一个可关闭的横幅：
> "检测到可能的紧急情况，需要查找附近 24 小时宠物急诊吗？"

- 点"查找" → 触发 Map Agent
- 点关闭 → 继续正常聊天
- 不打断对话，不替用户做决定

## SSE 流式响应事件类型

| 事件类型 | 前端行为 |
|----------|---------|
| `token` | 追加文字到当前 AI 气泡 |
| `card` | 渲染结构化卡片（record / map / email） |
| `emergency` | 显示急症提示横幅 |
| `done` | 结束流 |

## 首次使用流程

1. Apple / Google 登录（OAuth）
2. 首次登录后弹出免责声明弹窗，用户必须确认后才能使用
3. 引导用户添加第一只宠物（名字、品种、生日、体重、头像）

## 设计风格

- 极简、干净、温暖
- 适合宠物主人的柔和配色
- iOS 原生级的流畅动画和手势交互（左滑/右滑抽屉）
- 聊天气泡、卡片、横幅的设计要统一但有区分度

## 位置处理

- 位置权限按需请求（首次触发 Map Agent 时）
- 如无位置权限，Map Agent 提示用户分享位置或手动输入地址

## 认证

- JWT-based：access token（1h）+ refresh token（30d）
- 存储在 Capacitor Secure Storage（iOS Keychain）
- 请求通过 `Authorization: Bearer <token>` 传递
