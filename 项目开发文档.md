# PetPal — 宠物智能管家 产品文档

---

## 一、产品定位

PetPal 是一款以**对话为唯一交互方式**的宠物健康管家 App。

市面上的宠物 App 还在用传统的表单打卡，而 PetPal 让用户通过自然对话，自动生成结构化的健康日历、获取问诊参考、发现周边宠物友好场所。

**核心理念：** 这不是一个宠物聊天机器人，而是一个有记忆的宠物管家。聊天是输入方式，日历和时间线才是用户长期留存的价值。

## 三、核心功能

### 1. 智能对话（主功能）

用户打开 App 只看到一个聊天界面。说什么都可以——AI 自动理解你想做什么：

| 用户说的话 | AI 做的事 |
|-----------|----------|
| "Doudu vomited this morning" | 记录到健康日历 + 给出参考建议 |
| "Help me note down: vaccine on March 20" | 提取预约信息，写入日历 |
| "Any dog parks nearby?" | 推荐附近宠物友好地点 |
| "Summarize today's chat to calendar" | 回顾整段对话，提炼要点存入日历 |
| "Help me write an email to the vet" | 自动生成预约邮件，带症状时间线 |
| "Is it normal for cats to sneeze a lot?" | 给出参考建议（附免责声明） |

### 2. 健康日历（核心价值沉淀）

- 所有对话中的健康数据自动归档到日历
- 按日期查看每一天的记录
- 多宠物用颜色区分（橙色=狗狗，蓝色=猫咪）
- 支持手动编辑——AI 记错了可以直接改
- **两种写入方式：**
  - 显式记录：用户说"帮我记下来"，AI 立刻提取写入
  - 对话总结：聊完后用户说"总结到日历"，AI 回顾整段对话提炼写入

### 3. 智能提醒

- **定时提醒：** 驱虫到期、疫苗续打、预约就医
- **趋势洞察：** "最近一周记录了 3 次呕吐，建议关注"
- 通过 iOS 推送通知送达，不需要打开 App 也能收到
- 由 AI 在总结对话时自动标记哪些内容需要推送

### 4. 周边推荐

- 接入地图服务，推荐附近的宠物公园、宠物友好餐厅、宠物店、24 小时急诊
- AI 理解用户需求——说"周末想带狗户外"，推荐的是公园而不是宠物店
- 结果以卡片形式展示，包含名称、距离、评分

### 5. 一键生成预约邮件

- 聊完症状后，用户说"帮我写个邮件给兽医"
- AI 自动整理宠物信息、症状时间线，生成专业的预约邮件
- 用户可以在聊天中要求修改，满意后复制或分享

### 6. 多宠物管理

- 支持添加多只宠物（名字、品种、生日、体重、头像）
- 不需要切换宠物——AI 从对话上下文自动判断在说哪只
- "Doudu vomited" → AI 知道是 Doudu
- 说不清楚时 AI 会追问："你说的是哪只宝贝？"
- 日历上用不同颜色区分

## 四、用户界面

### 极简设计——只有一个页面

```
┌──────────────────────────────┐
│ 📅                        ⚙️ │
│                              │
│       对话流                  │
│                              │
│   ┌──────────────────────┐   │
│   │ AI建议仅供参考        │   │
│   └──────────────────────┘   │
│                              │
│  ┌────────────────────────┐  │
│  │  输入消息...        🎤  │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

- **没有底部导航栏，没有复杂菜单**
- 左上角 📅 → 滑出日历
- 右上角 ⚙️ → 滑出设置（宠物管理、账号、通知）
- 语音输入：点击麦克风调用系统键盘语音，自动转文字

### 对话中的智能卡片

AI 回复不只是文字，还包含：
- **记录卡片** — "已记录：3/17 Doudu 呕吐 黄水"，可点击跳转日历
- **地图卡片** — 附近地点列表，带名称、距离、评分
- **邮件卡片** — 生成的预约邮件，可复制/分享
- **急症提示** — 检测到危险关键词时弹出横幅（非强制，可关闭）

### 设计风格

- 暖色调 Warm Organic 风格（奶油底色 + 陶土橙点缀）
- 柔和圆润，温馨宠物感
- 未来支持用户自选主题配色

## 五、安全与急症处理

### 急症提示（非拦截）

当用户提到抽搐、中毒、误食、大出血、呼吸困难等关键词时：
- AI **正常回复**，不打断对话
- 聊天顶部弹出一个可关闭的提示横幅：
  > "检测到可能的紧急情况，需要查找附近 24 小时宠物急诊吗？"
- 用户点"查找" → 自动搜索附近急诊
- 用户忽略 → 继续聊天

**不替用户做决定，不强行拦截。**

### AI 安全规则

- 不做诊断，不开药
- 每次问诊回复结尾带免责声明
- 只提供参考建议，引导用户去看兽医

## 七、用户旅程

### 首次使用

1. 打开 App → Apple / Google 一键登录
2. 弹出免责声明 → 确认
3. 引导添加第一只宠物（名字、品种、生日、体重、头像）
4. 进入聊天页，开始对话

### 日常使用

```
用户："Doudu今天早上吐了黄水"
  ↓
AI 回复参考建议 + 自动记录到日历
  ↓
用户继续聊（或切换话题）
  ↓
用户："总结一下今天的内容放到日历"
  ↓
AI 回顾对话 → 提炼要点 → 写入日历 → 标记需要提醒的项目
  ↓
3天后推送通知："Doudu 最近一周呕吐 2 次，建议关注"
```

## 八、运营成本预估

以月活 **1,000 用户**，每人每天 **5 次对话**计算：

| 项目 | 月成本 |
|------|--------|
| AI 对话（强模型） | ~$50 |
| AI 提取/总结/路由（轻量模型） | ~$10 |
| 地图推荐（Google Places + 缓存） | ~$160 |
| 数据库 | ~$20 |
| 推送通知 | $0 |
| **合计** | **~$240/月** |

**成本控制策略：**
- 对话用强模型保证质量，其他功能用轻量模型省成本
- 地图结果缓存几小时，减少 API 调用
- 模型可随时切换更便宜的替代方案

## 九、竞争优势

| 维度 | 传统宠物 App | PetPal |
|------|------------|--------|
| 记录方式 | 手动填表打卡 | 聊天自动提取 |
| 用户粘性 | 低（只在需要时打开） | 高（日常聊天 + 管家提醒） |
| 数据沉淀 | 零散记录 | 结构化日历 + 趋势分析 |
| 问诊 | 无或独立模块 | 对话中自然融入 |
| 周边推荐 | 独立页面 | 对话中智能推荐 |
| 学习成本 | 需要了解各功能模块 | 零——会打字就会用 |
